# Система контроля и планирования ремонта МВПС (VKR)

REST API сервис и панель управления для автоматизации процессов технического обслуживания и ремонта (ТО/ТР) моторвагонного подвижного состава. 

Проект решает бизнес-задачи железнодорожного депо: управление ремонтными заданиями, контроль работы вспомогательных бригад и строгий учет расхода запчастей.

## 🛠 Технологии и стек

**Backend:**
* Python 3.10+
* FastAPI (веб-фреймворк)
* Pydantic (валидация данных и сериализация)

**Базы данных:**
* PostgreSQL (основная БД)
* SQLAlchemy (ORM)
* Alembic (миграции)

**Интерфейс и Администрирование:**
* SQLAdmin (панель управления)

**Инфраструктура и DevOps:**
* Docker & Docker Compose
* Linux / WSL2

## ⚙️ Функционал и Ролевая модель
В системе реализовано разделение прав доступа и два специализированных рабочих места, адаптированных под бизнес-процессы депо:

1. **Рабочее место "Мастер":** 
   Упрощенный и интуитивно понятный интерфейс, предназначенный для оперативного контроля. Позволяет мастеру быстро просматривать текущие ремонтные задания, заказывать материалы и отмечать статус выполнения без перегрузки лишней информацией.

2. **Рабочее место "Инженер" (SQLAdmin):** 
   Полноценная панель управления для аналитики, планирования и заведения ТО/ТР, учета номенклатуры запчастей, назначения бригад и приемки результатов ремонта. 

## 🏗 Архитектурные особенности
* **Изоляция бизнес-логики:** Несмотря на использование `SQLAdmin` для рабочего места "Инженера", запросы не идут напрямую в базу данных. Все действия из админки маршрутизируются через выделенный **сервисный слой (Service Layer)**.
* **Строгая валидация:** Данные, поступающие от пользователей или из панели администратора, в обязательном порядке проходят валидацию через схемы `Pydantic`, что гарантирует целостность базы данных и защиту от некорректного ввода.
* **Слоистая архитектура:** Проект придерживается принципов чистого кода с четким разделением на роутеры, сервисы и репозитории для работы с БД.

## 🚀 Инструкция по развертыванию

### 1. Системные требования
Для успешного развертывания инфраструктуры на хост-машине (сервере) должны быть установлены:
* **Операционная система:** Linux, macOS или Windows (через WSL2).
* **Docker:** Engine версии 20.10.0 и выше.
* **Docker Compose:** плагин версии v2.0.0 и выше.
* **Сеть:** Свободные порты `8000` (приложение/панель управления) и `5432` (PostgreSQL).

### 2. Структура конфигурационных файлов
Проект использует переменные окружения для настройки базы данных и параметров безопасности. Перед запуском необходимо создать файл `.env` в корневой директории репозитория.

**Пример конфигурации (`.env`):**
```ini
# Настройки подключения к СУБД PostgreSQL
DB_USER=postgres
DB_PASSWORD=your_strong_password
DB_HOST=db
DB_PORT=5432
DB_NAME=vkr_repair_db

# Секретный ключ для подписи токенов и сессий авторизации
SECRET_KEY=your_super_secret_key_here

3. Запуск проекта
Приложение полностью контейнеризировано. Для запуска базы данных и бэкенда выполните команду в корне проекта:

docker-compose up -d --build

После успешного запуска приложение и панель управления будут доступны по адресу: http://localhost:8000
